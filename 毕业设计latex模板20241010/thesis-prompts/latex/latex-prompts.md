# LaTeX 专用提示词

## 1. 正文转 LaTeX

```text
请将以下毕业论文正文整理为规范的 LaTeX 格式。

要求：
1. 使用 \section、\subsection、\subsubsection 组织层级；
2. 保持自然段完整，避免口语化表达；
3. 若涉及列举，优先使用 itemize 或 enumerate；
4. 不要使用 Markdown 语法；
5. 不要输出导言区，仅输出正文部分；
6. 不改变原有专业含义。
```

## 2. 图题与引导语

```text
请为以下论文插图设计学术化的图题与正文引导语。

要求：
1. 图题符合本科毕业论文规范；
2. 正文中使用“如图~\ref{fig:xxx} 所示”这类表达；
3. 图题简洁准确，正文引导语自然；
4. 适用于系统架构图、业务流程图、数据库 E-R 图、模块结构图、测试结果图等场景。
```

## 3. 表题与分析语

```text
请为以下论文表格撰写规范的表题和表后分析文字。

要求：
1. 使用正式、严谨的学术表达；
2. 表题准确概括表格内容；
3. 表后分析应体现设计意义或数据规律，不要只是重复表格内容；
4. 适用于角色权限表、数据库表结构、测试结果表、功能模块对照表等。
```

## 4. 摘要生成

```text
请根据“基于 Spring Boot 与 Vue 3 的企业供应链协同管理系统的设计与实现”这一论文题目，撰写中文摘要。

要求：
1. 摘要应包括研究背景、系统目标、技术路线、主要功能、实现效果和实际意义；
2. 控制在 300~500 字左右；
3. 体现 Spring Boot、Vue 3、JWT、RBAC、WebSocket、MySQL 等技术；
4. 突出多角色协同、库存预警、周计划生成、质量追溯等特点；
5. 最后给出 3~5 个关键词；
6. 输出 LaTeX 格式。
```

## 5. 致谢生成

```text
请撰写本科毕业论文“致谢”部分。

要求：
1. 风格真诚、庄重、得体；
2. 感谢指导教师、学院老师、同学与家人；
3. 避免过度口语化与夸张表达；
4. 控制在 400~600 字左右；
5. 输出 LaTeX 友好文本。
```
