# 项目事实说明

> 建议在每次章节写作时，将本文件与 `00-master-prompt.md` 一起提供给 Copilot。

```text
【项目名称】
基于 Spring Boot 与 Vue 3 的企业供应链协同管理系统

【系统定位】
本系统是一个面向企业业务协同的供应链管理系统，采用前后端分离架构实现。系统围绕订单驱动、库存联动、生产协同、采购供应协同和质量追溯等关键场景展开，覆盖顾客、销售管理员、仓库管理员、生产管理员、采购管理员、供应商、质检管理员以及系统管理员等多类角色。

【后端技术栈】
- Java 17
- Spring Boot 2.7
- Spring Security
- Spring Data JPA
- MySQL
- Maven

【前端技术栈】
- Vue 3
- Vite
- Axios
- SockJS
- STOMP

【关键技术机制】
- 基于 JWT 的身份认证机制
- 基于角色的访问控制机制（RBAC）
- 基于 WebSocket/STOMP 的实时消息推送
- 基于 JPA 的实体映射与数据持久化

【主要角色】
- 系统管理员
- 顾客
- 销售管理员
- 仓库管理员
- 生产管理员
- 采购管理员
- 供应商
- 质检管理员

【主要业务模块】
1. 用户认证与账号管理
2. 顾客下单与订单管理
3. 销售审核与销售记录管理
4. 仓库库存管理、出入库管理、库存预警
5. 生产计划下发、生产任务执行、生产记录管理
6. 采购申请、采购订单、供应商接单与发货、仓库收货入库
7. 质检处理与质量记录管理
8. 顾客质量追溯
9. 实时消息通知
10. 生产周计划与采购周计划自动生成

【系统流程特点】
- 顾客下单后进入销售审核流程；
- 销售审核通过后，订单进入仓库处理；
- 仓库根据库存情况决定直接发货或触发生产；
- 生产完成后进入质检流程；
- 质检合格后由仓库继续处理入库与发货；
- 原材料采购可由采购管理员主动发起，也可由库存预警触发采购申请；
- 供应商可接收采购通知并执行发货；
- 顾客可根据订单号进行质量追溯。

【工程实现亮点】
1. 多角色协同与权限隔离相结合；
2. 订单、库存、生产、采购、质检形成业务闭环；
3. 实时通知提升跨角色协同效率；
4. 库存预警与周计划生成增强决策支持能力；
5. 质量追溯增强系统透明度与客户信任度。

【适合论文重点展开的内容】
- Spring Security + JWT 的认证授权实现
- 多角色协同业务流程设计
- 库存预警与生产/采购计划联动
- WebSocket/STOMP 实时消息机制
- 质量追溯与质检记录留痕
- 周计划自动生成的工程实现思路
```
